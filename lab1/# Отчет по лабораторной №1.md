# Задание



Выделить на GPU массив arr из 10^9 элементов типа float и инициализировать его с помощью ядра следующим образом: arr\[i] = sin((i%360)\*Pi/180). Скопировать массив в память центрального процессора и посчитать ошибку err = sum\_i(abs(sin((i%360)\*Pi/180) - arr\[i]))/10^9. Провести исследование зависимости результата от использования функций: sin, sinf, \_\_sin. Объяснить результат. Проверить результат при использовании массива типа double.

&nbsp; Compute Capability: 7.5

&nbsp; Max Threads per Block: 1024

&nbsp; Clock Rate (kHz): 1545000

### Результат выполнения программы:

Error using \_\_sinf (float): 8.01664e-08 | Time: 19.2553 ms

Error using sinf (float): 8.3571e-09 | Time: 18.7772 ms

Error using sin (double): 8.77963e-18 | Time: 43.977 ms



### Выводы:

**1. Точность вычислений:** Тип double обеспечивает большую точность благодаря использованию 64-битного представления чисел с плавающей запятой по сравнению с 32-битным представлением для float. Это позволяет double представлять большее количество значащих цифр и уменьшать ошибки округления. В контексте вычислений, особенно при работе с тригонометрическими функциями, это приводит к более точным результатам, поскольку double может лучше сохранять информацию о значениях, которые обрабатываются.

**2.Время выполнения:** float (32 бита) требует меньше вычислительных ресурсов и памяти по сравнению с double (64 бита), что делает операции с float более быстрыми на GPU.
